# Disaster Recovery Runbook

## Цель

Обеспечить воспроизводимое восстановление после сбоев DB/Redis/приложения и зафиксировать процесс DR-проверок.

## Целевые показатели

- `RPO`: до 15 минут для core данных.
- `RTO`: до 60 минут для полного восстановления core контуров.

## Контур восстановления

1. Остановить запись в систему (maintenance mode/ограничение write endpoints).
2. Зафиксировать состояние и проверить целостность последних backup-артефактов.
3. Восстановить PostgreSQL из последнего валидного backup.
4. Прогнать миграции и health checks.
5. Поднять backend + восстановить Redis-зависимые сервисы.
6. Выполнить smoke-regression:
   - login;
   - создание заказа;
   - dispatch;
   - admin read-model endpoints;
   - offline sync push/pull.
7. Снять maintenance mode после подтверждения.

## Рекомендуемая периодичность DR drills

- Минимум 1 полный drill в месяц.
- После каждого существенного изменения схемы БД — внеплановый drill.

## Backup checklist

- Есть свежий backup PostgreSQL.
- Проверена возможность восстановить backup на тестовом окружении.
- Backup хранится в защищенном хранилище с retention policy.
- Доступ к backup ограничен по принципу least privilege.
